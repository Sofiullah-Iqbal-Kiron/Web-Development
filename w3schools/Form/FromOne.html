An HTML form is used to collect user input and sends to the server for further processing. An form element is a container for different types of input elements like buttons, text fields, checkboxes, radio buttons etc. The default width of an input field is 20 characters.

Form Attributes

  • action="file_name.ext" attribute defines the action to perform when the form is submitted. After submitting, the form data is sent to a file on the server. That file contains a server-side script to handle those data. If the action attribute is omitted, default action is set to redirect the current page.
  • target="_self | _blank" attribute specifies where to display the content that is returned by the server as a response against a request that was triggered when a user submitted the corresponding form.
  • method="GET | POST | dialog" attribute specifies the HTTP(isn't HTTPS more secure?) method to be used when submitting the form. Default value is GET. Never use GET to send sensitive data. The submitted form data is visible in the URL. From data appended to the action URL with a ? separator. On the other hand, POST method is processed by the server. Appends form-data inside the body of the HTTP request. That's why it's more secure to hide sensitive data.

HTML <select> tag:

HTML <select> tag creates a drop-down list. required attribute: the user is required to select a value from the drop-down list before submitting the form. The <option> tag defines an option in a <select>/<optgroup>/<datalist> and value attribute will sent to the server for backend processing.
